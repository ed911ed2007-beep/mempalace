# 強目標格式規則

## 規則

目標必須是「可驗證的」而非模糊的許願。

## 為什麼

- 弱目標無法確認完成與否
- 強目標有明確的驗收標準
- 減少來回確認，提高效率

## 怎麼套用

### 弱目標 → 強目標轉換

| ❌ 弱目標 | ✅ 強目標 |
|----------|----------|
| 幫我整理程式碼 | 用 const 取代所有 var，確保通過 ESLint |
| 幫我寫文件 | README 需包含：安裝步驟、使用範例、API 說明 |
| 優化效能 | 載入時間從 3s 降到 1s 以下 |
| 修 bug | 修復 issue #123，附上測試案例 |

## 反例

- 「讓程式碼更好」→ 無法驗證
- 「幫我看一下」→ 沒有明確產出
