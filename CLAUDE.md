# CLAUDE.md — mempalace 個人化設定

## 寫碼行為規則

### Rule 8 — 寫前先讀
新增或修改任何檔案前，必須先讀取該檔案的輸出介面、直接呼叫者與相關共用工具。
若不清楚現有結構為何如此，先問再動手。

### Rule 3 — 手術式修改
只改與需求直接相關的範圍。不去「順手改善」旁邊未損壞的程式碼或格式。

### Rule 1 — 思考再寫碼
實作前先明確說出假設。有任何模糊地帶，立刻停下來發問，不自行猜測。

### Rule 7 — 衝突攤開講
若程式庫中存在兩種相衝的寫法，選較新或測試較完整的那種，說明原因，
並標記另一種待清理。禁止混合寫法。
